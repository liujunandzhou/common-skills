它是怎么工作的

skill 实体始终留在 submodule 里（repo/ 下），不复制、不二次维护。 .claude/skills/ 里每个条目只是一条相对软链接，指回 submodule 内部的 skill 目录。 Claude Code 扫描 .claude/skills/ 时顺着链接读到 SKILL.md，skill 即可触发。

怎么用

因为 skill 实体在 submodule 里，clone 时必须递归拉取，否则软链接会指向空目录：

# clone 时带上 submodule
git clone --recurse-submodules git@github.com:liujunandzhou/common-skills.git

# 已经 clone 但漏了 submodule
git submodule update --init

# 升级某个 skill 包到上游最新
git submodule update --remote repo/pm-skills
在仓库里启动 Claude Code 后，/reload-skills 即可识别全部 71 个 skill，用 /<skill-name> 触发（例如 /create-prd、/html）。

再加一个 skill 包

# 1. 作为 submodule 加到 repo/ 下
git submodule add <repo-url> repo/<name>

# 2. 把其中的 skill 目录软链接到 .claude/skills/（相对路径）
cd .claude/skills
for d in ../../repo/<name>/path/to/skills/*/; do
  ln -sfn "$d" "$(basename "$d")"
done

约束：.claude/skills/ 下的目录名需全局唯一，避免跨包重名冲突。
